Brief Description of System

Las principales modificaciones con la anterior entrega, son las siguientes: - Ahora decidimos que se manda el contrato al usuario para que este pague y el sistema reciba la percepción de “pagado” del banco. - El actualizador de datos ha sido eliminado. Ahora los mismos obtenedores almacenan los datos que los usuarios piden en memoria para poderlos servir (mientras son vigentes) sin llamar a la api. - Separamos los agentes gestor de actividades y alojamiento, ya que entendemos que no nos beneficiamos de juntarlos. Eso es porque creemos que es mejor recaptar todos los elementos y hacer la acción selectiva en el agente planificador (donde tenemos todos los elementos para tal tarea) que no hacerlo por separado. - El plan se guarda solo cuando se ha aceptado.
